## What this project is

This is a supermarket price-comparison platform. Its main flow is:

- catalog stores products, stores, promotions, and price history.
- scraping collects listings from retailers with store-specific scrapers.
- The pricing engine computes the best “optimal unit price” for promos such as 2x1, 3x2, percent-off, and bulk deals.
- web exposes a dashboard and JSON APIs so you can compare prices, inspect history, and optimize a cart.
- celery.py and tasks.py schedule scraping and retention jobs with Celery/Redis.

The design is documented in ARCHITECTURE.md.

## How to start it

### 1) Prerequisites
You will need:
- Python
- PostgreSQL
- Redis
- optionally Chrome/Chromium for Playwright-based scraping

### 2) Create and activate a virtual environment

```powershell
cd "c:\Proyectos\supermarket tracking"
python -m venv venv
.\venv\Scripts\activate
```

### 3) Install dependencies

```powershell
pip install -r requirements.txt
pip install playwright
playwright install chromium
```

### 4) Prepare PostgreSQL and Redis
Make sure PostgreSQL is running and create a database, for example:

```powershell
createdb price_compare
```

If your setup requires a user and password, create them accordingly. Then make sure Redis is running, for example:

```powershell
redis-server
```

### 5) Wire the Django project
The repo includes the settings guidance in settings_snippet.py, but you still need to merge that into a real Django settings file and provide the Django entrypoint used by your environment. Once that is in place, the intended bootstrap flow is:

```powershell
python manage.py migrate
python manage.py pgpartition --yes
python manage.py seed_demo
python manage.py runserver
```

The seed command is defined in seed_demo.py, and it populates demo products and prices so the dashboard has data to show.

### 6) Open the app
After the server starts, open:

```text
http://127.0.0.1:8000/
```

## How to test it

### Basic smoke test
Open the dashboard in the browser and confirm it loads.

### API smoke test
Run these in another terminal:

```powershell

curl "http://127.0.0.1:8000/api/products/7790895000123/compare/"
```

### Cart API test

```powershell
curl -X POST "http://127.0.0.1:8000/api/cart/optimize/" `
  -H "Content-Type: application/json" `
  -d "{\"items\":[{\"ean\":\"7790895000123\",\"quantity\":2}]}"
```

## Optional: run the background jobs

If you want the scheduled scraping pipeline to work too, start Celery worker and beat:

```powershell
celery -A config.celery worker -l info
celery -A config.celery beat -l info
```
